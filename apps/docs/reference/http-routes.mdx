---
title: HTTP Routes and Discovery
description: Reference for the auto-mounted HTTP endpoints exposed by redop.
---

When you run the HTTP transport, `redop` mounts these routes automatically.

## Routes

| Method | Path | Purpose |
| --- | --- | --- |
| `POST` | `/mcp` | JSON-RPC endpoint |
| `GET` | `/mcp` | SSE stream for server-to-client events |
| `DELETE` | `/mcp` | Terminate an MCP session |
| `GET` | `/mcp/health` | Health check |
| `GET` | `/mcp/schema` | OpenAPI-style tool manifest |

## Why `/mcp/schema` matters

It gives you a generated view of registered tools and their input schemas, which is useful for debugging and discovery.

## Identity

The server identity returned during `initialize` comes from:

```ts
new Redop({
  name: "my-mcp-server",
  version: "0.1.0",
})
```
